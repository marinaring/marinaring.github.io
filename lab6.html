Lab 6: The Internet of Things and Serial Peripheral Interface

Introduction

In this lab, I learned how to work with and debug different communication protocols, specifically the Serial Peripheral Interface (SPI) communication protocol in this lab. We used SPI to create communication between a microcontroller and a DS1722 temperature chip. Then, the readings of the chip were displayed on an interactive website communicating with a ESP8266 WiFi development board. It was really cool to be able to connect all these different systems and see them working in harmony. See the E155 Lab 6 Page for a more detailed overview of the specifications and instructions for this lab.

All code written for this lab can be found on my E155 Lab 6 Repository on Github.


Setup and Circuitry

The ESP8266 WiFi development board was able to be plugged in directly onto female pin headers soldered onto my circuit board. The DS1722 temperature chip was connected directly through the breakout board. A schematic showing the pin connections is shown on the left and a picture of the actual setup is shown on the right.
